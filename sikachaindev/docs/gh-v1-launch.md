# Ghana v1 wallet launch checklist

Accra-first **Sika App** rollout: cGHS-first daily wallet + business MSIG, with pro tools hidden from hubs (direct URLs still work).

## 1. Environment (production build)

Set in the hosting provider / CI build env (never commit secrets):

| Variable | Ghana v1 value |
|----------|----------------|
| `NEXT_PUBLIC_MARKET` | `gh` |
| `NEXT_PUBLIC_WALLET_ROLLOUT` | `gh-v1` |
| `NEXT_PUBLIC_CHAIN_ID` | SikaChain mainnet/testnet chain ID |
| `NEXT_PUBLIC_CHAIN_READ_RPC_URLS` | HTTPS RPC endpoints (comma-separated) |
| `NEXT_PUBLIC_CONTRACT_ACCOUNT` | `sika` |
| `NEXT_PUBLIC_TABLE_ACCOUNT` | `sika` |
| `NEXT_PUBLIC_TOKEN_CONTRACT` | `sika.token` |
| `NEXT_PUBLIC_STABLECOIN_SYMBOL` | `CGHS` |
| `NEXT_PUBLIC_MSIG_CONTRACT` | `sika.msig` |
| `NEXT_PUBLIC_EXPLORER_HYPERION_URL` | Production Hyperion v2 base URL |
| `NEXT_PUBLIC_WEBSITE_URL` | GTM site (public explorer redirect) |

**Do not set** on production:

- `NEXT_PUBLIC_DEV_WALLET=1`
- `NEXT_PUBLIC_E2E_MOCK_SESSION=1`
- Dev private keys in any `NEXT_PUBLIC_*` var

### Sign-in (pick at least one)

| Method | Config |
|--------|--------|
| **Anchor mobile** | Default — no extra env |
| **Browser sign-in** | `NEXT_PUBLIC_WEB_AUTHENTICATOR_URL=https://…` (hosted [WharfKit web authenticator](https://github.com/wharfkit/wallet-plugin-web-authenticator)) |

Local preview of gh-v1 surface:

```bash
bash scripts/preview-gh-v1.sh   # sync env + copy .env.sikachaindev.gh-v1 → Sika app/.env.local
cd "/path/to/Sika app" && npm run dev
```

Production env template (placeholders — do not commit secrets):

- Sika app: `.env.production.gh-v1.example` (generated by `sync-dev-env.sh`)
- Anchor wallet import: `sikachaindev/anchor-chain.json` (from `export-anchor-chain.mjs`)

```bash
bash scripts/sync-dev-env.sh   # refreshes app env, adapter, GTM chain-constants, anchor-chain.json
```

## 2. What gh-v1 hides (hubs only)

Still available via direct URL / bookmarks:

- Swap quick action and More → Swap
- Wallet Tools hub (RAM, rent, producer, …)
- Explore power tools (contract console, name bids, …)

**Visible in gh-v1:** Send, Receive, Earn, Activity, Business (MSIG), Vote, core Explore (account, chain redirect, activity).

See `Sika app/docs/UI.md` and `src/app/_constants/walletRollout.ts`.

## 3. Pre-deploy verification (dev chain)

With SikaChainDev running and app on `:3003`:

```bash
bash scripts/check-launch-ready.sh        # templates + phase 3 (if RPC up)
LIVE=1 bash scripts/check-launch-ready.sh # + gh-v1 Playwright

# Full live wallet gate (20 tests: UI + on-chain send + MSIG + business import)
bash scripts/test-wallet-live.sh

# Ghana v1 rollout gating (2 tests, isolated :3099)
bash scripts/test-wallet-gh-v1.sh

# Both (22 tests)
GH_V1=1 bash scripts/test-wallet-live.sh

# Or from Sika app/
npm run test:wallet:live
npm run test:wallet:live:gh-v1   # 21 tests (live + gh-v1)
npm run test:wallet:gh-v1
npm run verify:gh-v1
```

On-chain MSIG CLI smoke (no browser):

```bash
bash scripts/ensure-wallet-app.sh      # fix stale :3003 dev server before Playwright
bash scripts/verify-msig-business.sh
bash scripts/cleanup-msig-proposals.sh   # optional — clear stale dev proposals
```

Full stack gate (Spring + contracts + Playwright):

```bash
export SIKACHAIN_DEV=1 SIKA_SYSTEM_ACCOUNT=sika
VERIFY_UI=1 VERIFY_DEV=1 bash scripts/verify-stack.sh
```

## 4. Chain prerequisites (mainnet / testnet)

- [ ] `sika.msig` deployed and **privileged**
- [ ] `sika.token` with **SIKA** + **CGHS**
- [ ] Hyperion v2 indexing SHIP from block producers
- [ ] RPC CORS allows wallet origin (or use same-origin API proxy)
- [ ] Savanna finality enabled (Spring with IF mode)

## 5. Post-deploy smoke (production)

Manual or automated against deployed URL:

1. Install PWA / open `/app/home`
2. Connect with Anchor (or web authenticator)
3. Home shows **cGHS** balance
4. Send 0.01 CGHS to a test account
5. Activity lists the transfer (Hyperion)
6. Business → propose → approve → exec on a test MSIG account
7. Confirm **Swap** and **Tools hub** are absent from More menu (gh-v1)

## 6. Rollback

Set `NEXT_PUBLIC_WALLET_ROLLOUT=full` (or unset) and redeploy to restore full hub surface without code changes.
